# CSV Template Types — TC FE

Template specs for all 3 CSV export types generated by `tc-fe-prep-workflow`.  
Source templates provided by user 2026-06-17.

Used by: `skills/deprecated/tc-fe-prep-workflow/WORKFLOW.md`

## Overview

| Type | Language | Columns | Row structure | Summary footer | Typical row cap |
|------|----------|---------|---------------|---------------|-----------------|
| **Unit Test** | English | 13 | Function/SubFunction groupings | No | Open-ended |
| **Integration Test** | Thai | 9 | Sequential numbered (blank) | Yes | ~330 |
| **System Test** | Thai | 9 | Sequential numbered (blank) | Yes | ~800 |

All types generate **blank templates** — row numbers and structure only, content columns empty.

## Template 1 — Unit Test

**Filename:** `Unit_Test_{ISSUE_KEY}.csv`

### Columns (13, English)

```
No., Sub Function, Test Scenario, Test Description, Pre-condition, Test Step, Test Data, Expected Result, Actual Result, Test status, Test Date, Test By, Comment
```

### Row structure

Has **section header rows** mixed in (not data rows):

| Row type | Example | Purpose |
|----------|---------|---------|
| Function header | `Function : <function name>` | Groups test cases by TOR function |
| Sub Function header | `Sub Function : <sub-feature name>` | Groups within a function |
| Data row | `1, <empty x12>` | Numbered TC row (content blank) |

Header rows: only col 1 populated, remaining 12 columns empty.  
Data rows: numbered sequentially within each sub-function group.

### No summary footer

## Template 2 — Integration Test

**Filename:** `Integration_Test_{ISSUE_KEY}.csv`

### Columns (9, Thai)

```
ลำดับ, โมดูล, รายการทดสอบ, ขั้นตอนการทดสอบ, ผลการทดสอบที่คาดหวัง, ผลการทดสอบ, วันที่ทดสอบ, ผู้ทดสอบ, หมายเหตุ
```

### Row structure

- Sequential numbered rows from 1 to N (columns 2–9 empty)
- No section grouping headers
- Typical N: **330 rows**

### Summary footer (after data rows)

Appended after the last numbered row, separated by 1 blank row:

```
,,,,รวม,0,,,
,,,,ผ่าน,0,,,
,,,,ไม่ผ่าน,0,,,
,,,,การทดสอบ (%),#DIV/0!,,,
,,,,ผ่านการทดสอบ (%),#DIV/0!,,,
```

Footer: column 5 = label, column 6 = value, columns 1–4 and 7–9 empty.

### Encoding

UTF-8 BOM (`utf-8-sig`) — required for Thai columns to open correctly in Excel.

## Template 3 — System Test

### Columns (9, Thai)


### Row structure

- Sequential numbered rows from 1 to N (columns 2–9 empty)
- No section grouping headers
- Typical N: **800 rows**

### Summary footer

Identical to Integration Test footer (same 5 rows, same columns).

### Encoding

UTF-8 BOM (`utf-8-sig`) — same as Integration Test.

## Python snippets

### Summary footer (Integration + System)

```python
writer.writerow([""] * 9)  # blank separator row
summary = [
    ["", "", "", "", "รวม", "0", "", "", ""],
    ["", "", "", "", "ผ่าน", "0", "", "", ""],
    ["", "", "", "", "ไม่ผ่าน", "0", "", "", ""],
    ["", "", "", "", "การทดสอบ (%)", "#DIV/0!", "", "", ""],
    ["", "", "", "", "ผ่านการทดสอบ (%)", "#DIV/0!", "", "", ""],
]
for row in summary:
    writer.writerow(row)
```

### Section header rows (Unit Test)

```python
def write_function_header(writer, function_name: str, n_cols: int = 13):
    writer.writerow([f"Function : {function_name}"] + [""] * (n_cols - 1))

def write_subfunction_header(writer, subfunction_name: str, n_cols: int = 13):
    writer.writerow([f"Sub Function : {subfunction_name}"] + [""] * (n_cols - 1))
```
